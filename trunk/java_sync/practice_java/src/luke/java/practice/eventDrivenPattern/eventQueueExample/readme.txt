事件在触发后并非立即得到广播，在将事件通知到各个监听器之前，将事件排入事件队列中。
之所以这样做，是因为如果软件流程(过程)很复杂，需要在状态达到相对稳定之后，才能处理事件。

将请求处理过程划分为若干个阶段，在完成每个阶段的处理后，框架进入相对稳定的状态，这时，检查事件队列，将其中的事件广播出去。

利用事件队列，我们可以在代码中人为地产生某类事件。这种能力，有时候是需要的。
如，假设在页面上有三个按钮，一个“保存”按钮，一个“发送”按钮和一个“保存并发送”按钮。
如果已经编写了“保存”按钮和“发送”按钮的动作事件处理方法，那么我们没有必要在“保存并发送”按钮中再重复一次代码，只需模拟“保存”和“发送”按钮产生的事件即可。
