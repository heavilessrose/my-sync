用回调方法响应事件。

与监听器类不同，回调方法不是一个类，而是类的一个方法。这个方法所属的类，可以是任何类，而不必是监听器类。所谓回调，是与正常调用相反的调用方向。在正常调用中，程序员编写的方法，调用操作系统或者类库中的方法。而回调正好相反，类库中的方法调用程序员编写的方法。

用回调方法响应事件，程序员只需编写一个方法，而不是一个完整的类，因此，比用监听器类响应事件要简单得多。

将回调方法指定为某事件的事件处理方法，其操作如同将监听器类注册到事件源一样.

值得说明的是，用回调方法来响应事件，并没有违背Java事件驱动的一般原理，在回调方法的背后，仍然是监听器在工作。

处理过程:
动作事件->通知缺省的监听器->执行processAction()->调用回调方法
