This library is the core, the foundation, over which the rest of the OpenBaseMovil projects are built.

It provides many resources needed for database or resource handling, and a lot of features present in JavaSE but no in JavaME.

  • Event management
    A centralized event manager that handles different types of events through the system, allowing multiple listeners globally or by event type. With custom event types, and event filtering.
  • File system
    Replacements for classes such as FileOutputStream or RandomAccessFileInputStream, but using record stores as the storage system.
  • Serialization
    The Serializable interface is a replacement of the missing java.io.Serializable, that allows any kind of object to be serialized and deserialized to/from a stream.
    It also contains streams for complex object serialization/deserialization, including complex trees of objects, with any mixture of Hashtables, Vectors, and other objects.
  • Logging
    A basic logging api that resembles that of the Jakarta Commons Logging package, with plugable implementations for debuging and production environments.
  • Math
    Contains the FixedPoint class, that replaces the missing Float/Double classes in CLDC 1.0.
  • MVC model
    Contains the basis of a Model-View-Controller system, taking care of such things as processing commands in a new Thread, and advanced concepts such as Actions, integrated help management, and much more.
  • Internationalization
    Allows you to use multiple languages in your applications, using resource files. Language is detected automatically, but can be changed manually on the fly. It has also a plugable architecture, so you can use any kind of resource storage, not just properties files.
    It also handles dates, numeric formats, currencies, etc.
  • Compression
    An adapted and shorter version of the jzlib library, adapted from zlib.
  • Other tools:
    • CRC, through the Adler32 algorithm.
    • Symetric encryption with Blowfish (from bouncycastle.org)
    • Message digest with MD5 (from bouncycastle.org)
    • Device extended information
    • Properties files support, for reading and for writing.
    • Simple cache system, with LRU discarding policy
